Swift заметки:


Опциональные типы.
1. ? - на самом деле призывает в этот момент nil
2. Если присвоить переменной с ? Какое-либо число, то оно будет в optional контейнере
3. Чтобы извлечь из контейнера, можно воспользоваться у переменной добавить ! , но это, если число не будет nil, потому что в другом случае - программа упадет. Это называется force unwrap
4. Для того, чтобы безопасно извлечь, можно воспользоваться Optional Binding: if let myWallet = wallet {}
5. Если переводить из строки в Integer, то результат будет Optional, потому что не факт, что в строке число, может быть string
6. Если необходимо присвоить короткой записью Optional число к обычному Integer, то можно воспользоваться x ?? 0. Если не удастся присвоить x значение, то тогда присвоиться 0 
Set:
1. Не сохраняется порядок элементов, но и не может быть дубликатов
2. Есть функция insert, чтобы добавить новое значение в Set. Если надо проверить, есть такое значение в Set или нет, то можно приписать вначале if и в конце .inserted {} и если есть, то выполнится один из блоков 
3. Чтобы удалить - .remove, после удаления - выводится Optional(и число), если числа не было, то nil
4. .contains(число) - для проверки, есть число или нет
5. .count - для подсчета количества элементов
6. .isEmpty, for in, 
7. .sorted - делает сортированный массив из этого сета
8. Для работы с двумя сетами есть метод Union(сет) - чтобы объединить два сета в один. Intersection(сет) - чтобы вывести похожие числа, которые есть и в первом сете и во втором. Subtracting(сет) - вывод из первого сета уникальные значения, которых нет во втором сете. SymmetricDiffrenece - создает сет только из уникальных значений, без повторных из двух сетов. 
9. Также, есть isSubset, если указать ему сет, и все значения первого сэта присутствуют во втором, то выводится true. isSuperset - это как isSubset, но наоборот, здесь второй сет на месте первого и обратно. isDisJoint - проверяет общие значения у первого сета, если нет - то false, если отсутствует, то true.

Guard If:
1. Guard y !=0 else { return } - проверяет условие, если не равно нулю - можно пользоваться функцией, если равно 0 - то сразу выходит из функции
2. Для установления диапазона, нужно x…y

Tuples (Кортежи)
1. К примеру: human = ("Alex”,10,0.5) и затем, можно присвоить массово каким-либо значениям let (name,age,height) = human. Но можно и обратиться по индексу - human.0, human.1. 
2. Можно объявить сразу имена параметрам: let human = (name: “Alex”, age: 30, height: 1.80), print(human.age)
3. Можно сделать функцию для вывода tuple. Func getHuman()->(String,Int,Double) { return(name:”Alex”, age: 30, height: 1.80) } Затем, использовать эту функцию: let human = getHuman() print(human.0)
4. К предыдущему пункту, можно только имя вывести, к примеру: let (name, _, _) = getHuman() , а затем print(name) . Это для того, чтобы место в памяти не занимать

Enum(перечисления)
1. Можно создать какое-либо перечисление, enum TypeDevice {case a case b} и создать константу let device:TypeDevice = .a
2. У enum есть классная фишка, ассоциативные значения (связанные значения с каким-то кейсом)

Classes (Классы)
1. Swift - ООП язык. Содержит объекты(экземпляры классов и структур)
2. Можно указать список свойств у класса. Это var\let
3. Если просто объявить свойства, но не присвоить какие-либо значения, то будет ошибка. Если присвоить значения, то они по умолчанию будут с этими значениями выводиться
4. Для создания экземпляра класса - let transport_one = Transport()
5. Если задавать обязательные свойства нет необходимости, то можно сделать их опциональными(присвоить nil), либо создать конструктор. Конструктор - функция, которая создает экземпляр класса и заполняет все обязательные свойства (init)
6. Subclass -> SuperClass
7. Если сделать Subclass, то чтобы с него передавать значения Superclass’a(у которого был объявлен init), то нужно в init обращаться через super.init и передавать туда значения (но нужно так же и в обычном init объявить их, после этого получится длинная гусеница в init, а в super.init меньше).
8. Дочерние классы могут переопределять методы родительского. Для этого достаточно написать название функции или самому вручную, вписав override func название функции()

Структуры
1. В отличии от классов, в структурах сразу автоматически создается init
2. Структуры не могут наследоваться, в отличии от классов
3. Если мы меняем свойство экземпляра структуры, то нам необходимо указать методу вначале - mutating
4. Структуры - value type, в отличии от классов(которые ссылочные)
5. Let - не перезаписывается, но можно внутри функции новый экземпляр структуры объявить изменить там нужные значения и вывести его
